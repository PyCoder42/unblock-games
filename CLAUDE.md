# CLAUDE.md

This file is the **single source of truth** for any agent working on this repository. Read it fully before touching code.

## What This Is

HTML game files served over local WiFi to bypass school network restrictions. Each game lives in its own folder inside `Games/` with protocol-managed file variants. A remote config system (jsDelivr CDN + GitHub raw + unpkg fallback) enables password management and per-game blocking from a phone via the admin panel. A Val Town auth server adds server-side security (password checking, IP blocking, version invalidation) so that no secrets or bypass logic live in the client HTML.

## Architecture

### File Variant System

Each game folder inside `Games/` contains these HTML files:

- **`*-regular.html`** (base) — Clean game HTML, no protocol metadata. Source of truth.
- **`*-open-in-new-tab.html`** — Lightweight wrapper that opens the game in a new tab. For code editors like hcodx.com.
- **`*-secure.html`** — Outer HTML is an opaque base64 blob. Inner HTML (decoded at runtime) has the password UI and either AUTH_URL mode (POST to Val Town) or legacy CDN mode. View-source reveals nothing useful.
- **`*-proxy.html`** — Standalone proxy launcher. Opens the game through the Val Town proxy. Uses shared `proxy_worker_url` localStorage key.

All variants except proxy are generated from the base file by `lock-game.sh`. The `sync-protocol-versions.sh` script batch-generates across all game folders.

**Note:** The `*-locked.html` variant was phased out. Default keep set is `base,secure`.

### Remote Config System

**GAS (Google Apps Script) has been fully removed.** Two modes exist:

#### Auth Server Mode (preferred — when AUTH_URL is set in .secure-config)

Secure files POST to the Val Town auth server. The server checks password, game blocking, IP blocking, and file version server-side. On approval, it returns the encrypted game data + decryption key. The client HTML contains NO repo URLs, NO encryption keys, NO blocking logic — just a password prompt and the AUTH_URL.

Flow: `User enters password → POST to AUTH_URL/auth → server fetches config from GitHub (server-side) → checks everything → returns encrypted game + key → browser decrypts and loads`

#### Legacy CDN Mode (fallback — when AUTH_URL is not set)

Secure files fetch config from a CDN fallback chain:

1. **jsDelivr CDN** (primary) — `cdn.jsdelivr.net/gh/REPO@<tag>/config.json`
2. **GitHub raw** (fallback) — `raw.githubusercontent.com/REPO/main/config.json`
3. **unpkg** (3rd fallback) — `unpkg.com/@scope/pkg@latest/config.json`
4. **Auto-block** — If ALL fail, `showBlocked()` fires.

**Write flow:** Admin panel writes to GitHub API → creates a version tag → purges jsDelivr cache.

### Val Town Auth Server (`Proxy/valtown-auth.ts`)

Server-side security layer deployed on Val Town (val.run domain, reachable at school).

**Endpoints:**
- `POST /auth` — Check password, game blocking, IP blocking, file version. Returns encrypted game data + decryption key on success.
- `GET /config` — Read current config (requires AUTH_SECRET header).
- `POST /config` — Write config (requires AUTH_SECRET header).
- `POST /block-all` — Set `minFileVersion` to now, invalidating all current secure files.
- `POST /block-ip` — Add IP to blockedIps.
- `POST /unblock-ip` — Remove IP from blockedIps.

**Environment variables (set in Val Town):** `GITHUB_PAT`, `GITHUB_REPO`, `GAME_KEYS_JSON`, `AUTH_SECRET`

### Val Town Web Proxy (`Proxy/valtown-proxy.ts`)

Serverless web proxy deployed on Val Town. Fetches blocked websites server-side, rewrites internal links to route back through the proxy. School Chromebooks can navigate to val.run (confirmed via `tests/proxy-host-tester.html`).

**Featured sites:** Wikipedia, Pokemon Showdown, GN Math, Cool Math Games, Smash Karts.

**Shared localStorage key:** All proxy game files store/read the proxy URL from `localStorage.proxy_worker_url`. Enter the URL once, all games remember it.

### Encrypted Game Data (`game-data/`)

Secure variants do NOT embed game HTML inline. Instead:

1. `lock-game.sh` encrypts game HTML with AES-256-GCM via `generate-enc-game-data.mjs`
2. Encrypted payload stored in `game-data/<game-id>.enc.json`
3. In auth mode: key returned by server. In legacy mode: key embedded in secure HTML.
4. At runtime, `fetchAndDecryptGame()` fetches encrypted data from jsDelivr and decrypts in-browser.

Keys stored in `.game-keys` (gitignored). This folder **must stay at repo root** — jsDelivr serves `game-data/<id>.enc.json` by path.

### Config npm Package (`config-pkg/`)

The unpkg fallback is backed by this folder:

- `config-pkg/package.json` — npm package metadata
- `config-pkg/config.json` — config copy for unpkg
- `Formatting Scripts/publish-config-npm.sh` — copies config, bumps version, runs `npm publish`

This folder **must stay at repo root** — the npm package publishes from here.

### config.json (root)

The remote config source file. Schema:

```json
{
  "password": "string",
  "passwords": ["string"],
  "blocked": { "gameId": true | "ISO_date" },
  "games": ["gameId", ...],
  "allowedIps": { "ip": { "label": "string" } },
  "blockedIps": { "ip": { "label": "string", "blockedAt": "ISO_date" } },
  "minFileVersion": 0
}
```

- `blockedIps` — IPs that are blocked from accessing any game (managed via admin panel or auth server).
- `minFileVersion` — Timestamp. Any secure file with `FILE_VERSION < minFileVersion` is auto-blocked. Used by the "Block All Versions" admin button to invalidate all current files.

This file **must stay at repo root** — jsDelivr serves `config.json` by path.

### Admin Panel System

- **Real panels** have a GitHub PAT with write access. Full control.
- **Fake panels** have their own fine-grained GitHub PAT. Revoking the PAT = total lockout.
- **Template:** `Formatting Scripts/admin-panel-template.html` with placeholders filled by `setup-secure.sh`.
- **Template variables:** `{{JSDELIVR_URL}}`, `{{GITHUB_RAW_URL}}`, `{{UNPKG_URL}}`, `{{GITHUB_REPO}}`, `{{GITHUB_PAT}}`, `{{PANEL_MODE}}`, `{{AUTH_URL}}`.
- **`.secure-config` and `Admin Panel/` contents are gitignored.**
- **Admin panel features (planned):** "Block All Versions" button (POST to AUTH_URL/block-all), IP block/unblock UI (POST to AUTH_URL/block-ip and /unblock-ip).

### Proxy System

**Platform:** Val Town (val.run domain reachable at school; workers.dev is BLOCKED).

**Two Val Town vals needed:**
1. **Proxy val** (`valtown-proxy.ts`) — Web proxy for browsing blocked sites.
2. **Auth val** (`valtown-auth.ts`) — Security server for game file authentication.

**Game proxy files** (in each game folder):
- `coolmath-game-proxy.html` — Loads Cool Math Games through proxy
- `smash-karts-proxy.html` — Loads Smash Karts through proxy
- `pokemon-showdown-proxy.html` — Redirects to Pokemon Showdown through proxy

All share the `proxy_worker_url` localStorage key — enter the proxy URL once, all remember it.

**Pokemon Showdown migration:** Once the Val Town proxy is deployed, Pokemon Showdown will use it instead of the current Cloudflare Tunnel approach (`start-pokemon-showdown-remote.sh` + `pokemon-showdown-host.js`).

### Smash Karts Technical Details

Unity WebGL game requiring special handling:

- **jsDelivr 20MB limit:** Build files exceed 20MB. Solution: `Pok12d/ta` repo splits into two zip parts (~17MB each).
- **Loading pipeline:** JSZip → download zip parts → combine → extract to blob URLs → URL interception → Unity loader.
- **Monkeypatch architecture:** `document.open()/write()/close()` replaces the page, losing DOM patches. Fix: inject all monkeypatches INSIDE the written HTML as a top-of-page `<script>`.
- **Mocks required:** Firebase, PokiSDK, gtag/dataLayer (all inline).
- **Assets CDN:** `cdn.jsdelivr.net/gh/Pok12d/ta@main/sma/`

### CDN Accessibility (School Chromebook)

Tested via `tests/cdn-tester-regular.html` and `tests/proxy-host-tester.html`:

| Status | Services |
|--------|----------|
| **CORS OK** | jsDelivr, cdnjs, unpkg, esm.sh, Skypack, Google CDN, Microsoft CDN, GitHub raw |
| **Reachable (no CORS)** | Val Town (val.run), Fly.io, Render, Vercel, Netlify |
| **Blocked** | Cloudflare Workers (workers.dev), Statically, GitHack, Cloudflare Pages, Heroku (intermittent) |

"No CORS" = direct navigation works (good for proxy), but `fetch()` from another page doesn't.

### GN-Math Pattern

Multi-repo structure: `zones.json` catalog in `gn-math/assets`, game HTML files in `gn-math/html`, covers in `gn-math/covers` — all via jsDelivr. Games fetched as text and written into iframe via `contentDocument.open()/write()/close()`. Main repo DMCA'd by Poki (Dec 2025) but asset repos still work. Local `gn-math-regular.html` uses `<base href>` to resolve against CDN.

## Key Conventions

- **"base" vs "-regular"**: Scripts use `base`; files on disk use `-regular.html`. See `docs/LOCKING_PROTOCOL.md`.
- **Eaglercraft naming**: Version suffix after variant kind: `eaglercraft-regular-1-12.html`, `eaglercraft-secure-1-12.html`.
- **Banner markers**: `<!-- LOCK-BANNER START/END -->`. Version tracked by `__LOCK_BANNER_V4__`.
- **Fullscreen persistence**: Fullscreen active during password → game loads in iframe. Otherwise, page replaced.
- **Default password**: `supercoolpassword`
- **jsDelivr versioning**: Semver tags (`0.0.<timestamp><random>`) not branch refs.
- **FILE_VERSION**: Timestamp embedded in secure files at generation time. Compared against `config.minFileVersion` for version invalidation.
- **IP display**: Secure files show the user's IP at the bottom of the login screen (clickable to copy).

### Architecture Decisions Log
- **Why jsDelivr over GitHub raw?** jsDelivr not blocked at school; GitHub.com is.
- **Why AES encryption over base64?** Base64 is trivially decodable. AES-256-GCM with CDN-hosted data means view-source reveals nothing.
- **Why Val Town over Cloudflare Workers?** workers.dev is blocked at school. val.run is reachable.
- **Why auth server over client-side checks?** Client-side HTML exposes repo URLs, encryption keys, and blocking logic — all bypassable. Auth server keeps secrets server-side.
- **Why version tags over branch refs?** jsDelivr metadata lags behind branch HEAD. Exact tags are immediately available.
- **Why no GAS?** GAS was blocked on the school network.
